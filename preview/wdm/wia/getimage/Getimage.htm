Sample WIA Application

SUMMARY

This sample shows how to access Windows Image Acquisition (WIA) from an application and can also be used to test WIA user-mode minidrivers. It is a command line program which can list the WIA devices installed, list the items available from a device, and transfer an item from a device. Getimage can be used with the sample drivers or with real drivers.

At the time of this writing, WIA is only supported on Windows Me. This sample will not work on Windows 98 or Windows 2000.

BUILDING THE SAMPLE

  1. In the Start menu, select  "Free Build Environment" or "Checked Build Environment" under "Windows DDK" to open a command window and set basic environment variables needed to build the application.
  2. Change to the directory containing the sample source code, such as cd src\preview\wdm\wia\getimage.
  3. Execute the "build" command. After the build completes, an executable named Getimage.exe will be located in a subdirectory.

USING THE SAMPLE

To use the sample application, first install a WIA device such as the sample scanner or camera. Getimage.exe is run directly from the command line. Use the “/?” switch to get a full list of options. The following are the most common commands:

·        /enum – Enumerates the devices installed

·        /items – Recursively enumerates all of the items available from the device
